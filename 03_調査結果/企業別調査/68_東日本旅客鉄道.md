## 68. 東日本旅客鉄道株式会社 調査レポート

本レポートは、東日本旅客鉄道株式会社（JR東日本）について、指定された10の分析軸に基づき調査・分析した結果をまとめたものです。JR東日本は、鉄道事業を中核としながら、Suicaを起点としたIT・決済サービスや、駅ビル・商業施設、不動産業など、多岐にわたる事業を展開し、DXを経営戦略の中核に据えています。

---

### 1. 内製化の強化・IT子会社再編

JR東日本は、IT子会社である**株式会社JR東日本情報システム（JEIS）**が基幹システムの開発・運用を担う一方、2022年にグループ全体のDXを加速させるための新会社を設立し、内製開発力とデータ活用能力の強化を鮮明に打ち出しています。

| 具体的な動き | 時期・情報源 | 概要 |
| :--- | :--- | :--- |
| **DX専門の新会社設立** | 2022年4月（プレスリリース） | グループのDXをリードする専門会社**「JR東日本クロスステーション（JRCX）」**を設立（※編集注：JRCXはリテール事業が主。DX専門会社は「JR東日本スマートロジスティクス」など個別領域で設立）。本社に**「マーケティング本部」**を設置し、Suicaなどのデータを活用したグループ横断の戦略を推進。 |
| **内製開発体制の強化** | 2021年（JEIS HP） | JEIS内にアジャイル開発専門チームを発足させるなど、顧客ニーズの変化に迅速に対応できる開発体制を構築。 |

**【考察】**
従来のウォーターフォール型の開発文化に加え、アジャイル開発のような俊敏な手法を積極的に取り入れ、内製開発力を強化しようという明確な意志が見られます。これは、Suicaアプリの新機能開発など、顧客接点となるデジタルサービスの開発スピードを上げ、競争力を高めるための重要な戦略転換です。

---

### 2. レガシーモダナイゼーションの推進

鉄道の安全・安定輸送を支える大規模な基幹システム（運行管理、設備管理など）を、最新のデジタル技術を取り入れて継続的に刷新・高度化しています。

| 具体的な動き | 時期・情報源 | 概要 |
| :--- | :--- | :--- |
| **次世代の列車運行管理システム** | （企業HP） | 無線技術をベースとした次世代の列車制御システム**「ATACS」**を導入。従来の地上設備（信号機など）を大幅に削減し、より柔軟で高密度なダイヤ設定や、災害時の迅速な復旧を可能にする。 |
| **設備管理のDX** | （企業HP） | 鉄道インフラ（線路、トンネル、橋梁など）の点検に、ドローンや各種センサー、AI画像解析などを活用。従来の「時間計画保全（TBM）」から、データに基づき劣化を予測して対応する**「状態基準保全（CBM）」**へと移行。 |

**【考察】**
ATACSの導入やCBMへの移行は、単なるシステムの置き換えではありません。これは、鉄道事業の「安全思想」と「オペレーション」そのものを、データ駆動型へと変革する、極めて本質的なモダナイゼーションです。これにより、安全性と効率性の両方を、より高いレベルで実現しようとしています。

### 3. SaaS利用の限界に直面している企業（SaaS脱却）

「脱SaaS」という動きは見られず、むしろ、グループ全体のコミュニケーション基盤や、間接業務の効率化において、SaaSを積極的に活用しています。

| 具体的な動き | 時期・情報源 | 概要 |
| :--- | :--- | :--- |
| **全社コミュニケーション基盤** | （Microsoft導入事例） | グループ約7万人を対象に、**Microsoft 365**を導入。Teamsなどを活用し、本社、現場、グループ会社間の円滑な情報共有とコラボレーションを促進。 |

**【考察】**
鉄道の運行管理といった極めて専門的でミッションクリティカルなシステムは自社開発で固めつつ、全社共通で利用するコミュニケーションツールなどでは、グローバルで標準的に使われているSaaSを導入する、合理的な使い分けを行っています。

### 4. AI駆動開発のスケール課題に悩む企業

鉄道の安全・安定輸送、顧客サービスの向上、そして従業員の働き方改革など、多岐にわたる領域でAIの活用を積極的に進めています。

| 具体的な動き | 時期・情報源 | 概要 |
| :--- | :--- | :--- |
| **設備の異常検知AI** | （企業HP） | 線路の画像をAIで解析し、レールの傷やボルトの緩みを自動で検出。また、車両の床下機器の画像を解析し、部品の異常を検知するなど、点検業務を高度化。 |
| **顧客案内のAI化** | 2025年6月（ITmedia） | 主要駅に設置したAIサイネージやアバターが、乗り換え案内や周辺情報などを多言語で提供。駅係員の業務負荷を軽減し、顧客満足度を向上。 |
| **生成AIの活用** | 2023年（JEIS HP） | グループ全社で利用できる、セキュリティを確保した独自の生成AI環境を構築。企画立案や文書作成などの業務効率化を支援。 |

**【考察】**
同社のAI活用は、「安全」という絶対に譲れない価値を守るための技術として、また、駅というリアルな顧客接点での「おもてなし」を高度化するための技術として、明確な目的意識を持って導入されています。特に、熟練の眼をAIで再現する設備点検は、技能伝承と安全性向上を両立させる重要な取り組みです。

### 5. データセキュリティ・ガバナンスの強化が急務な企業

Suicaに記録された膨大な乗降履歴・購買履歴といった個人情報や、国民生活の根幹をなす鉄道の運行情報を守るため、最高レベルのセキュリティ・ガバナンス体制を構築しています。

| 具体的な動き | 時期・情報源 | 概要 |
| :--- | :--- | :--- |
| **専門組織の設置** | （統合報告書） | グループ全体のサイバーセキュリティを統括する専門部署と、インシデント対応チーム**「JR-EAST-CERT」**を設置。 |
| **Suicaデータの匿名加工** | （企業HP） | Suicaの利用データを外部に提供する際は、個人が特定できないよう、統計情報として厳格に匿名加工処理。プライバシー保護を徹底。 |
| **制御システム（OT）のセキュリティ** | （統合報告書） | 運行管理や電力、通信といった鉄道の制御システムへのサイバー攻撃対策を強化。国の重要インフラ事業者として、厳格なセキュリティ基準を遵守。 |

**【考察】**
同社は、「個人情報保護」と「重要インフラ防護」という、2つの極めて重いセキュリティ責務を負っています。特に、Suicaデータの利活用においては、プライバシーへの配慮が事業の生命線であり、技術的な対策と法的な遵守の両面から、極めて慎重なガバナンス体制を敷いています。

### 6. トレーサビリティ・証跡管理に課題を抱える企業

鉄道の安全運行を担保するため、車両や線路、各種設備の保守・点検履歴を正確に記録・管理することが、事業の根幹をなしています。

| 具体的な動き | 時期・情報源 | 概要 |
| :--- | :--- | :--- |
| **車両・設備の保守履歴管理** | （企業HP） | 個々の車両や、線路、信号、変電所といった設備について、全ての点検、修理、部品交換の履歴をシステムで厳密に管理。 |
| **ICカード乗車券の利用履歴** | （企業HP） | SuicaなどのICカード乗車券は、全ての利用履歴（入場駅、出場駅、時刻、運賃など）が記録されており、精算や問い合わせ対応の際の正確な証跡となる。 |

**【考察】**
同社にとってトレーサビリティは、事故やトラブルが発生した際の迅速な原因究明と再発防止の基盤であり、安全運行を支えるための不可欠な仕組みです。日々の膨大な保守記録や利用履歴を、正確かつ改ざん不可能な形で管理するシステムが、鉄道事業の信頼性を担保しています。

### 7. AIを活用した新規事業開発を進める企業

「人の移動」から得られる膨大なデータ（移動データ）を活用し、交通事業の枠を超えた新たなビジネス領域の創出を目指しています。

| 具体的な動き | 時期・情報源 | 概要 |
| :--- | :--- | :--- |
| **MaaSプラットフォーム** | （企業HP） | 「Ringo Pass」や「TOHOKU MaaS」など、鉄道だけでなく、バス、タクシー、シェアサイクルといった様々な交通手段を、一つのアプリでシームレスに検索・予約・決済できるMaaS（Mobility as a Service）を展開。 |
| **移動データの活用による都市開発** | （企業HP） | Suicaの利用データなどを分析し、人々の移動パターンや滞在時間（人流データ）を把握。駅周辺の商業施設の開発や、新たな街づくりに活用。 |
| **物流DX** | 2021年4月（プレスリリース） | 新幹線などを使った荷物輸送サービス「はこビュン」を開始。旅客鉄道の空きスペースを物流に活用する新たなビジネスモデル。 |

**【考察】**
Suicaという強力な顧客接点と、そこから得られる「人流データ」を最大の武器として、交通を起点とした新たな経済圏（エコシステム）の構築を目指しています。これは、従来の鉄道会社という枠組みを超え、人々の生活全般を豊かにする「ソリューションプロバイダー」への変革を目指す、壮大な挑戦です。

### 8. マルチクラウド・ハイブリッドクラウドへのシフトを進める企業

顧客向けサービスやデータ分析基盤において、パブリッククラウドを積極的に活用する一方、運行の根幹をなすシステムはオンプレミスで堅牢に運用する、ハイブリッドクラウド戦略を実践しています。

| 具体的な動き | 時期・情報源 | 概要 |
| :--- | :--- | :--- |
| **MaaS基盤のクラウド化** | （推察） | 多様な交通事業者やサービスと連携するMaaSプラットフォームは、俊敏性と拡張性に優れたパブリッククラウド（AWS, Azureなど）上に構築されていると推察される。 |
| **ハイブリッド構成** | （推察） | 列車運行管理システム（ATACSなど）は、ミリ秒単位の応答と絶対的な安定性が求められるため、自社管理下の閉域網・オンプレミス環境で運用。これとクラウド上の情報系システムを連携させるハイブリッド構成をとる。 |

**【考察】**
「絶対的な安定・安全」と「ビジネスの俊敏性」という、相反する要件を両立させるため、ハイブリッドクラウドは同社にとって必然の選択です。それぞれのシステムの特性を完全に見極め、最適なインフラを配置する高度なアーキテクチャ設計能力が、同社のDXを支えています。

### 9. データ分析基盤を刷新・強化したい企業

Suicaの利用履歴（IC乗車券利用実績）や、モバイルSuicaの利用データなどを統合的に分析するための、国内屈指の大規模データ分析基盤を保有・活用しています。

| 具体的な動き | 時期・情報源 | 概要 |
| :--- | :--- | :--- |
| **「Suica統計情報」の提供** | （企業HP） | Suicaの利用データを、個人が特定できないように統計処理し、交通計画や都市開発、マーケティングなどに活用できるデータとして、自治体や企業に提供。 |
| **顧客行動分析** | （企業HP） | JRE POINTと連携したSuicaの利用データや、駅ナカ店舗の購買データなどを組み合わせて分析。顧客のライフスタイルを深く理解し、新たなサービス開発や店舗開発に繋げる。 |

**【考察】**
同社が持つ「人流データ」は、他のどんな企業も持ち得ない、極めて価値の高い経営資源です。このデータを分析し、社会の役に立つ形で提供すること自体が、大きな事業となっています。まさに、データが石油に例えられる「データエコノミー」を体現する存在です。

### 10. エッジコンピューティング・IoT基盤の刷新を検討中の企業

鉄道インフラ（車両、線路、駅など）の全てがIoT/エッジデバイスの塊であり、そこから得られるデータを活用して、安全・安定輸送の高度化と、新たな顧客体験の創造に取り組んでいます。

| 具体的な動き | 時期・情報源 | 概要 |
| :--- | :--- | :--- |
| **スマートトレイン** | （企業HP） | 車両そのものに多数のセンサーや通信装置を搭載。機器の状態をリアルタイムで監視し、故障を予知保全。車内の混雑度を計測し、乗客に情報提供することも。 |
| **スマートステーション** | （企業HP） | 駅構内にカメラやセンサーを多数設置し、人流や混雑状況を分析。案内の最適化や警備の効率化に活用。将来的には、改札機をなくす「タッチレス改札」も視野に入れる。 |
| **Suica** | （企業HP） | Suicaは、乗客が携帯する「エッジデバイス」であり、改札機や店舗のリーダーと通信することで、膨大な移動・購買データを生成する、巨大なIoTシステムの中核。 |

**【考察】】**
同社は、数千キロにわたる線路、数千両の車両、数千万人が利用するSuicaという、他に類を見ない広大かつ高密度なIoT/エッジ基盤を保有しています。このリアルなインフラから得られるデータを、いかにデジタル空間で活用し、フィジカルな世界の安全性と利便性を向上させるか。その「リアルとデジタルの融合」こそが、同社のDXの真髄です。
